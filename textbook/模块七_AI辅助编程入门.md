## 第25课时 AI编程概述与环境搭建

### 章节导读

"编程"曾经是程序员的专属领域。但在AI时代，一位建筑学生也可以用AI一句话生成一个精美的网页、一个好用的小程序。本课时不要求你学习编程语法，而是帮你理解"AI辅助编程"的思维方式，并搭建好工具环境，让你能够在接下来的课时中亲手做出自己的数字作品。

### 一、为什么建筑专业学生需要了解AI编程？

很多同学会问：我是学建筑的，为什么要学编程？

答案很简单：**AI编程让你的设计作品可以"活起来"**。

| 传统表达方式 | AI编程赋能的新表达 |
|------------|-----------------|
| 打印作品集手册 | 在线作品集网站，随时更新 |
| PPT展示方案 | 交互式方案展示网页 |
| 微信发图片 | 专属小程序，品牌感更强 |
| 纸质展板 | 数字展览，可分享、可互动 |
| 静态效果图 | 带360°全景的网页展示 |

更重要的是，在AI辅助编程时代，**你不需要懂语法，只需要能描述你想要什么**。这和提示词工程的思路完全一致——你在模块二学到的提示词技巧，在这里同样适用。

> 🏛️ **样式雷精神的现代延伸**：样式雷家族用"烫样"（微缩模型）来呈现设计方案，让皇帝直观理解设计意图。AI编程让今天的建筑师能用网页、小程序来呈现方案，实现跨越时代的"数字烫样"。

### 二、主流AI编程工具介绍

#### 2.1 工具对比一览

| 工具 | 类型 | 特点 | 适合场景 | 费用 |
|------|------|------|---------|------|
| **Cursor** | 桌面IDE | 最强代码补全，完整开发环境 | 完整项目开发 | 免费+付费版 |
| **Claude Code** | 命令行工具 | Anthropic官方，理解能力强 | 复杂逻辑任务 | 按token计费 |

> 💡 **新手建议**：从**豆包（Trae）**或**Cursor**开始，界面友好，中文支持好，上手最快。

#### 2.2 AI编程的工作原理

AI编程工具本质上是**"理解你的意图 → 生成代码 → 帮你运行"**的流程：

```
你的想法（自然语言描述）
        ↓
AI理解意图，生成对应代码
        ↓
代码在本地或云端运行
        ↓
浏览器/手机中看到结果
        ↓
不满意？继续用语言描述修改需求
        ↓
AI再次修改代码
```

这个过程中，**你负责提需求和审美判断，AI负责写代码**。

#### 2.3 三个关键概念

在学习AI编程之前，需要了解三个基本概念：

**① 网页（Website）**
- 由HTML（结构）、CSS（样式）、JavaScript（交互）组成
- 运行在浏览器中，可以通过链接分享
- 建筑应用：个人作品集、项目展示页、公司官网

**② 小程序（Mini Program）**
- 运行在微信/支付宝等App内，无需下载安装
- 开发门槛比普通App低，传播方便
- 建筑应用：楼盘展示、施工管理、物业服务

**③ 代码编辑器（IDE）**
- 写代码的专用软件，类似Word之于写作
- AI辅助编程工具就集成在这里
- 常见的：VS Code、Cursor、微信开发者工具

## 第26课时 AI制作网页——个人作品集

### 章节导读

本课时的目标只有一个：**用AI做出一个真实可用的个人建筑作品集网页**。你不需要懂任何编程语法，只需要会描述你想要的效果。课时结束时，你将拥有一个可以发给别人查看的作品集链接。

### 一、用AI一句话生成网页

#### 1.1 最简单的开始

打开Cursor，按 `Ctrl+I` 打开AI对话，输入：

```
帮我创建一个个人建筑作品集网页。
要求：
- 深色风格（深灰色背景，白色文字）
- 页面顶部有名字和专业（城乡规划专业）
- 中间展示6个作品格子（可以先放图片占位）
- 底部有联系方式
- 要有专业感，像建筑师的作品集
请直接生成HTML文件。
```

AI会生成完整的HTML代码，保存为 `index.html`，用浏览器打开即可预览。

```
【图片占位】IMG-7-02
标题：AI生成的建筑作品集网页效果
内容：浏览器中显示的建筑作品集网页，深色风格，展示作品网格
尺寸：全宽，约1/2页高
来源：课程截图
```

#### 1.2 网页的基本结构

AI生成的网页代码通常包含三个文件：

```
my_portfolio/
├── index.html     ← 主页面（结构）
├── style.css      ← 样式文件（颜色、字体、排版）
└── script.js      ← 交互脚本（可选，用于动态效果）
```

你不需要懂代码，但要知道**修改哪里**：
- 想改颜色、字体 → 描述给AI，让它改 `style.css`
- 想改文字内容 → 直接在 `index.html` 里找到文字修改
- 想加动画效果 → 描述给AI，让它修改或新增 `script.js`

#### 1.3 逐步优化网页

生成初稿后，通过持续对话来优化：

**第一轮**：生成基础页面

**第二轮**：调整风格
```
请把网页改成更有设计感的风格：
- 导航栏加入Logo占位
- 作品图片加入悬停放大效果
- 加入作品分类标签（城乡规划/室内设计/景观设计）
```

**第三轮**：添加内容
```
在作品展示区，帮我添加以下项目信息：
- 项目名称：城市滨水空间改造设计
- 项目类型：城乡规划
- 完成时间：2024年
- 简介：针对城市河道沿线150米范围进行景观提升设计
```

**第四轮**：添加联系方式和下载按钮
```
在底部加入：
- 邮箱：xxx@xxx.com
- 一个"下载PDF版本"的按钮（按钮样式和主题色一致）
```

## 第27课时 AI制作小程序——建筑项目展示

### 章节导读

小程序是比网页更贴近日常生活的展示平台——用户无需下载App，扫码就能看。本课时将带你用AI制作一个建筑项目展示小程序，让你的设计方案可以通过微信二维码分享给任何人。

### 一、小程序基础知识

#### 1.1 什么是小程序？

微信小程序是运行在微信内部的轻量级应用。特点：
- **无需安装**：扫描二维码即可打开
- **天然传播**：通过微信直接分享
- **手机优先**：界面为手机屏幕优化
- **可发布**：审核通过后可供所有微信用户访问

对建筑专业学生来说，小程序适合做：
- 个人作品集展示
- 建筑方案展示（配合VR/全景图）
- 楼盘样板间展示
- 校园活动报名（综合实训展示）

#### 1.2 小程序文件结构

```
my_miniapp/
├── app.js          ← 全局逻辑
├── app.json        ← 全局配置（页面列表、导航栏颜色）
├── app.wxss        ← 全局样式
└── pages/
    ├── index/      ← 首页
    │   ├── index.js
    │   ├── index.wxml    ← 相当于HTML
    │   └── index.wxss    ← 相当于CSS
    └── detail/     ← 详情页
        ├── detail.js
        ├── detail.wxml
        └── detail.wxss
```

> 💡 不需要记住这些，AI会帮你处理所有文件。你只需要知道"有这样一个结构"。

## 第28课时 OpenClaw与移动端AI编程

### 章节导读

前几课时我们在电脑上完成了AI编程任务。但有没有可能在手机上就完成编程工作？OpenClaw正是为此而生——它让你可以随时随地用手机执行AI编程任务。本课时将介绍OpenClaw的特色功能，并通过实操体验移动端AI编程的魅力。

### 一、OpenClaw简介

#### 1.1 什么是OpenClaw？

OpenClaw是一款类似Claude Code的AI编程工具，最大的特色是：

- **手机端操作**：在手机上就能完成复杂的编程任务
- **手机通信功能**：可以读取手机消息、发送通知等
- **与Claude Code互补**：Claude Code更适合电脑端的大型项目；OpenClaw更适合移动场景和快速任务

#### 1.2 与Claude Code的对比

| 对比维度 | Claude Code | OpenClaw |
|---------|------------|---------|
| 运行平台 | 电脑端（命令行） | 手机端+电脑端 |
| 操作方式 | 文字命令 | 文字+语音 |
| 手机通信 | 不支持 | 支持 |
| 适合场景 | 大型项目开发 | 快速任务、移动场景 |
| 学习曲线 | 较陡（需要了解命令行） | 较平缓（接近普通App） |
| 费用 | 按token计费 | 按使用计费 |

### 三、移动端AI编程的技巧与限制

#### 3.1 移动端的优势

- **随时随地**：在工地、在甲方处、在通勤路上都能用
- **语音输入**：描述需求比打字更快
- **拍照理解**：拍下草图让AI识别并转化为代码

#### 3.2 移动端的限制

- 屏幕小，查看和调试代码不方便
- 复杂项目建议回到电脑端处理
- 网络要求较高

#### 3.3 最佳实践：手机+电脑协作

```
手机（现场）
  ↓ 语音描述需求/拍照上传草图
AI生成初版代码
  ↓ 代码发送到电脑（通过微信、邮件等）
电脑（精细调整）
  ↓ 用Cursor完善代码
最终作品
```
