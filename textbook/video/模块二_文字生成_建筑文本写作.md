## 第3课时 基础对话与学习辅助

### 章节导读

文字生成是AIGC最基础也最常用的能力。对于建筑专业学生而言，AI文字助手就像一位随时在线的"学习伙伴"——它可以帮你解读复杂的建筑规范、翻译英文文献、分析设计案例、整理学习笔记。本课时将教你如何与AI进行有效对话，让它真正成为你的专业学习助手。

### 一、与AI对话的基本方法

#### 1.1 认识对话界面

以DeepSeek为例，AI对话界面的核心要素包括：

```
┌──────────────────────────────────────────┐
│  DeepSeek                                │
│                                          │
│  ┌─ 历史对话列表 ─┐  ┌─ 对话主区域 ──┐  │
│  │ 新建对话        │  │               │  │
│  │ 昨天的对话1     │  │  AI的回答     │  │
│  │ 昨天的对话2     │  │  显示在这里   │  │
│  │ ...            │  │               │  │
│  └────────────────┘  │               │  │
│                      │               │  │
│                      ├───────────────┤  │
│                      │ 在此输入你的问题│  │
│                      │          [发送]│  │
│                      └───────────────┘  │
└──────────────────────────────────────────┘
```

**基本操作要点**：
- **新建对话**：每个新话题建议开一个新对话，避免上下文混乱
- **连续追问**：在同一对话中可以持续追问，AI会记住上下文
- **复制回答**：AI回答通常可以一键复制，方便粘贴到文档中
- **重新生成**：对回答不满意可以点击"重新生成"，AI会给出不同的版本

#### 1.2 提问的五个层次

与AI对话的质量，很大程度上取决于你"问"的水平：

| 层次 | 提问方式 | 示例 | 回答质量 |
|------|---------|------|---------|
| **L1 模糊提问** | 随意提问，没有具体方向 | "建筑设计怎么做？" | 回答笼统、泛泛而谈 |
| **L2 具体提问** | 有明确的问题 | "现代主义建筑有哪些特点？" | 回答准确但较基础 |
| **L3 场景提问** | 结合具体场景 | "我在做一个小学建筑设计，现代主义风格在学校建筑中有哪些适合的应用？" | 回答有针对性 |
| **L4 结构化提问** | 明确角色、要求和格式 | "你是一位建筑设计专家，请从造型、材料、空间三个维度分析现代主义风格在学校建筑中的应用，每个维度给出2个案例" | 回答专业、有条理 |
| **L5 系统化提问** | 多轮对话，逐步深入 | 先问概述→再问细节→追问案例→要求总结 | 回答全面、深入 |

> **关键原则**：你给AI的信息越具体，AI给你的回答就越有用。

#### 1.3 AI对话的典型场景

**场景1：知识学习**

```
示例对话：

用户：请用通俗易懂的语言解释什么是"容积率"，我是城乡规划专业的大二学生。

AI：容积率（FAR，Floor Area Ratio）简单来说就是：
    容积率 = 总建筑面积 ÷ 用地面积

    打个比方，一块1000平方米的地：
    - 盖1层、每层500平方米 → 容积率 = 500/1000 = 0.5
    - 盖10层、每层500平方米 → 容积率 = 5000/1000 = 5.0

    容积率越高，说明土地利用越密集。一般来说：
    - 别墅区：0.3-0.5
    - 多层住宅：1.0-1.5
    - 小高层：2.0-3.0
    - 高层住宅：3.0-5.0
    - 商业中心：5.0-10.0
    ......
```

**场景2：规范解读**

```
示例对话：

用户：请解读《建筑设计防火规范》GB50016-2014中关于住宅建筑
疏散楼梯设置的要求，重点说明不同建筑高度对应的楼梯形式。

AI：根据《建筑设计防火规范》GB50016-2014，住宅建筑疏散楼梯
的设置要求如下：

1. 建筑高度≤21m（约7层以下）：
   - 可采用敞开楼梯间
   ......

⚠️ 注意：AI对规范的解读可能存在偏差，请务必对照规范原文核实。
```

**场景3：翻译辅助**

```
示例对话：

用户：请翻译以下建筑英文摘要为中文，保留专业术语的英文原文
（用括号标注），并在翻译后列出关键专业词汇对照表：

"Sustainable architecture integrates passive design strategies,
including natural ventilation, daylighting, and thermal mass,
to minimize energy consumption while maintaining occupant comfort."

AI：可持续建筑（Sustainable architecture）整合了被动式设计策略
（passive design strategies），包括自然通风（natural ventilation）、
自然采光（daylighting）和蓄热体（thermal mass），以在保持居住者
舒适度（occupant comfort）的同时最大限度地减少能源消耗。

关键专业词汇对照表：
| 英文 | 中文 |
|------|------|
| Sustainable architecture | 可持续建筑 |
| Passive design strategies | 被动式设计策略 |
| Natural ventilation | 自然通风 |
| Daylighting | 自然采光 |
| Thermal mass | 蓄热体 |
| Energy consumption | 能源消耗 |
| Occupant comfort | 居住者舒适度 |
```

**场景4：案例分析**

```
示例对话：

用户：请从设计理念、空间布局、材料运用、景观融合四个方面
分析安藤忠雄的"光之教堂"。

AI：光之教堂分析：

一、设计理念
安藤忠雄的光之教堂（Church of the Light）位于日本大阪，
建于1989年。其设计理念的核心是"以最少的物质创造最深刻的
精神空间"......
```

**场景5：归纳总结**

```
示例对话：

用户：我在准备城乡规划原理考试，请帮我把以下笔记内容
整理成思维导图形式的文字大纲：[粘贴笔记内容]

AI：城乡规划原理复习大纲

一、城市规划基本概念
├── 1.1 城市的定义与特征
│   ├── 人口聚集
│   ├── 产业聚集
│   └── 空间组织
├── 1.2 城市化
│   ├── 定义
│   ├── 衡量指标
│   └── 中国城市化进程
└── 1.3 城市规划的概念
    ├── 定义
    ├── 目的
    └── 基本原则
......
```

### 二、高效对话的实用技巧

#### 2.1 让AI扮演特定角色

在提问前指定AI的身份，可以让回答更加专业和有针对性：

| 角色指令 | 适用场景 |
|---------|---------|
| "你是一位有20年经验的建筑设计师" | 咨询设计方案、审查设计文件 |
| "你是一位城乡规划专业的教授" | 学习规划理论、准备考试 |
| "你是一位室内设计公司的总监" | 讨论设计风格、材料选择 |
| "你是一位园林景观设计专家" | 植物配置、景观节点设计 |
| "你是一位建筑规范审查员" | 检查设计是否符合规范 |

#### 2.2 指定输出格式

告诉AI你想要什么格式的输出，比自由发挥效果好得多：

```
常用格式指令：
- "请用表格形式对比..."
- "请用分点列出..."
- "请用思维导图大纲形式整理..."
- "请控制在500字以内..."
- "请分为三个部分：概述、详解、总结"
- "请用Markdown格式输出"
```

#### 2.3 提供参考资料

当你需要AI处理特定内容时，直接把资料粘贴给它：

```
有效的做法：
"请根据以下设计任务书的内容，提取关键设计要求并整理成表格：
[粘贴设计任务书内容]"

低效的做法：
"帮我整理一下设计任务书的要求"（AI不知道你的任务书内容是什么）
```

#### 2.4 多轮对话深入探索

不要试图一次问完所有问题，善用多轮对话逐步深入：

```
第1轮：请概述绿色建筑评价标准的主要内容
第2轮：请详细说明"节能与能源利用"这个评分项的要求
第3轮：其中关于外墙保温的要求，常用的做法有哪些？
第4轮：在我们所在的夏热冬冷地区，推荐哪种保温做法？
第5轮：请帮我整理以上内容，输出一份学习笔记
```

## 第4课时 提示词工程

### 章节导读

上一课时我们学会了与AI进行基础对话。但你可能已经发现，有时候AI的回答不够专业、不够具体、格式不理想。这是因为你的"指令"——也就是**提示词（Prompt）**——还不够精确。本课时将系统学习提示词工程，让你能够精准控制AI的输出。

### 一、为什么需要提示词？

#### 1.1 一个对比实验

让我们用同一个问题，对比"没有提示词"和"有提示词"的效果差异：

**无提示词版本**：

```
用户：帮我写一个设计说明。

AI：好的，以下是一份设计说明：

本项目位于城市中心，总面积约5000平方米。设计理念以人为本，
注重功能分区的合理性。建筑采用现代风格，外立面使用玻璃幕墙
和石材相结合......

（问题：AI不知道是什么项目，只能胡编一个通用的、空洞的说明）
```

**有提示词版本**：

```
用户：你是一位有15年经验的室内设计师，现在需要为以下项目
撰写设计说明。

【项目信息】
- 项目名称：XX大学图书馆阅览室改造
- 面积：800平方米
- 设计风格：现代简约+新中式元素
- 功能要求：自习区、电子阅览区、休闲阅读区、研讨室
- 服务对象：在校大学生

【要求】
- 字数1000-1500字
- 包含设计理念、空间布局、材料选择、色彩方案四个部分
- 语言专业规范，适合设计方案汇报使用
- 融入中国传统书院文化元素

AI：（输出一份结构清晰、内容具体、专业规范的设计说明）
```

> **结论**：同样是AI，给出的提示词不同，输出质量天差地别。提示词就是你与AI沟通的"设计图纸"——图纸越详细，成品质量越高。

#### 1.2 什么是提示词工程？

**提示词（Prompt）**：你输入给AI的指令文本，包括问题、要求、背景信息等所有内容。

**提示词工程（Prompt Engineering）**：通过优化提示词的内容和结构，让AI产出更高质量内容的方法论。

简单来说：**提示词工程 = 学会"说话"让AI更好地为你工作**。

### 二、提示词七要素

经过大量实践总结，一个优秀的提示词通常包含以下七个要素：

#### 2.1 七要素总览

| 要素 | 说明 | 是否必需 | 建筑场景示例 |
|------|------|---------|-------------|
| **角色** | 给AI设定一个专业身份 | 推荐 | "你是一位注册建筑师" |
| **背景** | 提供项目或任务的背景信息 | 推荐 | "这是一个乡村振兴项目" |
| **事项** | 明确要AI做什么 | 必需 | "撰写设计说明书" |
| **要求** | 具体的质量和内容要求 | 推荐 | "包含设计理念、空间分析" |
| **样式** | 输出的格式和排版要求 | 可选 | "用Markdown格式，分章节" |
| **风格** | 语言风格和表达方式 | 可选 | "语言正式专业，适合评审" |
| **结构** | 输出的整体框架 | 可选 | "分为五个部分：总述、各论..." |

> **记忆口诀**：**角背事要，样风结**（角色-背景-事项-要求-样式-风格-结构）

#### 2.2 要素详解与示例

**要素一：角色（Role）**

给AI一个明确的身份定位，让它从特定的专业视角回答问题。

```
基础版：
"你是一位建筑设计师"

进阶版：
"你是一位在甲级设计院工作了15年的注册建筑师，
擅长教育建筑和文化建筑设计，
熟悉国家建筑设计规范和绿色建筑评价标准"
```

**为什么角色很重要？** 因为同一个问题，不同角色的回答侧重点完全不同：

| 角色 | 问"如何设计一个好的学校入口？" |
|------|------------------------------|
| 建筑师 | 强调空间序列感、尺度把控、立面标识性 |
| 规划师 | 强调交通组织、人车分流、与城市道路的衔接 |
| 室内设计师 | 强调门厅氛围、展示功能、导视系统 |
| 景观设计师 | 强调入口广场、绿化配置、文化景观节点 |

**要素二：背景（Context）**

提供必要的背景信息，让AI理解具体的项目语境。

```
示例：
"项目位于江西省南昌市红谷滩区，属于夏热冬冷气候区。
这是一个老旧小区改造项目，始建于1998年，
共有12栋多层住宅楼，住户约500户。
业主是当地街道办事处，预算有限，
需要在不影响居民正常生活的前提下分期施工。"
```

**为什么背景很重要？** 建筑设计高度依赖场地和环境条件，AI需要这些信息才能给出有针对性的建议。

**要素三：事项（Task）**

明确告诉AI你要它做什么，用动词开头。

```
常见事项动词：
撰写、分析、对比、总结、设计、规划、评估、翻译、
解释、列举、生成、优化、审查、推荐
```

```
示例：
❌ "关于室内设计色彩搭配"（不清楚要做什么）
✅ "分析北欧风格室内设计的常用色彩搭配方案"（明确动作）
```

**要素四：要求（Requirements）**

对输出内容的具体质量要求。

```
示例：
- 字数要求："控制在800-1000字"
- 内容要求："必须包含设计理念、空间布局、材料选择三个方面"
- 专业要求："引用相关国家标准和规范条号"
- 受众要求："内容面向非专业的业主，避免使用过多专业术语"
- 数量要求："列举5个以上具体案例"
```

**要素五：样式（Format）**

指定输出的排版格式。

```
常用格式指令：
- "用表格形式输出"
- "用编号列表形式"
- "用Markdown格式，包含标题和分级"
- "用对比形式，左边传统方法，右边AI方法"
- "先给结论，再展开论述"
```

**要素六：风格（Tone）**

指定语言的表达风格。

```
常用风格指令：
- "语言正式专业，适合评审报告"
- "通俗易懂，面向普通业主"
- "简洁干练，类似项目简报"
- "生动有趣，适合公众号推文"
- "学术严谨，适合论文写作"
```

**要素七：结构（Structure）**

预设输出的整体框架。

```
示例：
"请按以下结构撰写：
一、项目概况（200字）
二、设计理念（300字）
三、总体布局（300字）
四、建筑单体设计（300字）
五、景观环境设计（200字）
六、技术经济指标表格"
```

### 三、提示词实战模板

```

【要求】

2. [要求2]
3. [要求3]

【格式】[输出格式要求]

【风格】[语言风格要求]
```

#### 3.2 建筑领域专用模板

**模板A：设计说明书生成**

```
【角色】你是一位甲级设计院的资深建筑师/室内设计师/景观设计师。

【项目信息】
- 项目名称：____
- 项目地点：____
- 项目类型：____（住宅/商业/办公/学校/...）
- 建筑面积/用地面积：____
- 设计风格：____
- 功能要求：____
- 特殊要求：____

【任务】请撰写本项目的设计说明书。

【要求】
1. 包含设计理念、总体布局、功能分区、空间设计、材料选型、
   色彩方案等内容
2. 引用相关设计规范
3. 语言专业规范
4. 字数____字

【结构】
一、项目概况
二、设计理念
三、[根据具体项目类型调整]
四、......
```

**模板B：规范解读**

```
【角色】你是一位建筑设计规范审查专家。

【任务】请解读以下规范条文：[粘贴条文内容]

【要求】
1. 用通俗易懂的语言解释每一条的含义
2. 说明在实际设计中的注意事项
3. 举出常见的违规案例
4. 如有相关联的其他规范条文，请一并说明

【格式】每条规范的解读按"条文原文→通俗解读→设计注意事项→
常见问题"的结构输出。

⚠️ 提醒：规范条号和内容以官方文件为准，AI解读仅供参考。
```

**模板C：方案分析报告**

```
【角色】你是一位城乡规划专业的研究员。

【背景】[项目背景和分析目的]

【任务】请对以下项目/场地进行分析，撰写分析报告。

【分析维度】
1. 区位分析
2. 现状分析
3. SWOT分析
4. 案例借鉴
5. 规划策略建议

【要求】
- 每个维度需要具体数据支撑（可合理假设）
- 语言专业但不晦涩
- 配合文字，说明建议配什么样的分析图
- 字数2000-3000字
```

**模板D：投标书摘要**

```
【角色】你是一位建筑设计投标经验丰富的技术总监。

【项目信息】[招标项目的基本信息]

【任务】请撰写本项目投标文件的技术摘要部分。

【要求】
1. 突出设计亮点和创新点
2. 体现团队技术实力
3. 回应招标文件中的核心要求
4. 语言简洁有力，适合评委快速阅读
5. 字数500-800字
```

### 四、提示词优化技巧

#### 4.1 迭代优化法

提示词不必一次写到完美，可以通过多轮对话逐步优化：

```
第1轮：发送基本提示词，获得初步输出
第2轮："第二段太笼统了，请加入更多关于材料选择的具体说明"
第3轮："设计理念部分请融入地域文化元素"
第4轮："整体不错，但请把语言风格调整为更正式、更适合评审"
第5轮："请输出最终版本"
```

#### 4.2 "先思考再回答"技巧

对于复杂问题，可以要求AI先分析再输出：

```
"在回答之前，请先思考以下几个方面：
1. 本项目的设计重点和难点是什么？
2. 有哪些类似的成功案例可以借鉴？
3. 需要注意哪些规范要求？
然后再基于以上思考，撰写设计说明。"
```

#### 4.3 提供示例（少样本学习）

给AI一个你满意的范例，让它按照同样的风格和质量输出：

```
"请参考以下示例的写作风格和深度，为另一个项目撰写类似的
设计说明：

【示例】
项目名称：XX中学教学楼
设计说明：
本项目以'书院新境'为设计理念，将传统书院的空间精神
融入现代教育建筑......（完整示例文本）

【新任务】
请为以下项目撰写同样风格的设计说明：
项目名称：XX小学综合楼
......"
```

#### 4.4 让AI改进自己的提示词

你甚至可以让AI帮你优化提示词：

```
"我想用AI生成一份景观设计说明书，以下是我写的提示词，
请帮我优化这个提示词，让它能生成更高质量的内容：

[你的原始提示词]"
```

#### 1.2 AI输出评估要点

生成内容后，需要从以下几个维度评估和修改：

| 评估维度 | 检查要点 | 常见问题 |
|---------|---------|---------|
| **专业性** | 术语是否准确？材料推荐是否合理？ | AI可能推荐不存在的材料品牌 |
| **逻辑性** | 设计理念与具体做法是否一致？ | 理念说"极简"但做法很繁复 |
| **可行性** | 方案在实际施工中是否可行？ | 忽略结构限制、消防要求 |
| **创意性** | 是否有独特的设计亮点？ | AI容易生成同质化内容 |
| **规范性** | 是否符合相关设计规范？ | 疏散宽度、防火分区等 |

> **重要提醒**：AI生成的设计说明是"初稿"，必须经过专业人员审核修改才能使用。

## 第6课时 AI内容幻觉与信息验证

### 章节导读

经过前几课时的实操，你可能已经感受到AI的强大——它能在短时间内生成结构完整、语言流畅的专业文本。但与此同时，你可能也发现了一个严重的问题：**AI有时候会一本正经地胡说八道**。这就是"AI幻觉"（AI Hallucination）。对于建筑行业而言，如果不加辨别地使用AI生成的内容，可能导致引用错误的规范条号、使用不存在的材料数据、甚至给出违反安全要求的建议。本课时将深入剖析AI幻觉的本质，并教你如何有效识别和应对。

### 一、什么是AI幻觉？

#### 1.1 定义

**AI幻觉（AI Hallucination）**：AI生成的内容看起来合理、流畅、自信，但实际上包含虚假、错误或编造的信息。

通俗来说：**AI不会说"我不知道"，它宁可编一个听起来很像的答案**。

#### 1.2 AI幻觉的表现形式

| 幻觉类型 | 表现 | 建筑领域举例 |
|---------|------|-------------|
| **编造事实** | 捏造不存在的数据、事件 | "根据2024年住建部统计，中国绿色建筑面积达到XX亿平方米"（数据可能是编的） |
| **虚构引用** | 给出不存在的文献或规范条号 | "依据《建筑设计防火规范》GB50016-2014第5.3.7条..."（该条号可能不存在或内容不对） |
| **张冠李戴** | 混淆不同事物的信息 | 把安藤忠雄的作品说成是扎哈·哈迪德的 |
| **过度推断** | 从有限信息得出不可靠的结论 | "根据场地朝向，建议采用XX保温做法"（推断缺乏依据） |
| **自相矛盾** | 前后说法不一致 | 前面说"建议采用框架结构"，后面说"利用剪力墙提高抗震性能" |

#### 1.3 为什么AI会产生幻觉？

回到模块一学过的AI原理：大语言模型本质上是一个"超级联想机器"，它通过预测下一个最可能出现的词来生成文本。

```
AI的思维过程：
"建筑设计防火规范中关于..."
→ 下一个词最可能是"疏散"
→ "疏散...通道...宽度...不应小于..."
→ 统计上最可能的数字是"1.1米"
```

AI并不真正"理解"规范内容，也不会去"查阅"原文，它只是根据训练数据中的模式生成最可能的文本。当训练数据不足或模式匹配出错时，就会产生幻觉。

**三个根本原因**：

1. **训练数据有限**：AI的知识来自训练数据，如果某方面数据不足（如地方性建筑规范），它就容易编造
2. **模式匹配而非理解**：AI不理解建筑规范背后的工程逻辑，只是匹配文字模式
3. **"自信"的设计机制**：AI被设计为给出完整流畅的回答，而不是说"我不确定"

### 二、建筑领域典型幻觉案例

#### 2.1 案例一：编造规范条号

```
用户：请问住宅建筑的楼梯间最小净宽是多少？

AI（含幻觉的回答）：
根据《住宅设计规范》GB50096-2011第6.3.2条规定，
住宅建筑楼梯间的最小净宽不应小于1.10m。

⚠️ 核查结果：
- 条号可能不准确（实际规范中该条号对应的内容可能不同）
- 数值可能近似正确，但具体规定需要区分楼梯类型和住宅层数
- 正确做法：查阅规范原文核实
```

#### 2.2 案例二：虚构案例数据

```
用户：请列举3个获得中国绿色建筑三星级认证的学校建筑案例。

AI（含幻觉的回答）：
1. 北京XX大学图书馆（2019年获三星认证），建筑面积35000㎡...
2. 上海XX小学综合楼（2020年获三星认证），采用地源热泵系统...
3. 深圳XX中学教学楼（2021年获三星认证），年节能率达40%...

⚠️ 核查结果：
- 项目名称可能是编造的
- 获证年份和技术参数可能不准确
- 有些可能确实存在，但细节被AI"补全"了
```

#### 2.3 案例三：材料参数错误

```
用户：EPS保温板的导热系数是多少？

AI（含幻觉的回答）：
EPS保温板（膨胀聚苯板）的导热系数为0.041W/(m·K)。

⚠️ 核查结果：
- 数值大致在合理范围内（通常0.039-0.042）
- 但不同密度、不同厂家的EPS板导热系数不同
- 用于设计计算时必须以材料检测报告为准
```

### 三、如何减少AI幻觉的影响？

#### 3.1 六个实用策略

**策略一：要求AI标注不确定信息**

```
在提示词中加入：
"如果你对某些信息不确定，请明确标注'[待核实]'，
不要编造数据。对于规范条号，如果不确定，请注明
'建议核实原文'。"
```

**策略二：提供参考资料**

```
"请基于以下规范原文回答问题，不要超出原文内容范围：

[粘贴规范原文]

问题：......"
```

> 这种方式称为**RAG（检索增强生成）**，是目前减少幻觉最有效的方法之一。

**策略三：要求引用来源**

```
"请在回答中标注每个关键信息的来源（规范名称、条号、
或参考文献），如果无法确定来源，请说明。"
```

**策略四：交叉验证**

```
方法：将同一个问题分别输入2-3个不同的AI工具，
对比它们的回答。如果回答一致，可信度较高；
如果回答不同，需要人工核实。
```

**策略五：限制AI的发挥空间**

```
"请仅基于以下已知信息回答，不要添加任何额外假设：
[列出已知信息]"
```

**策略六：人工审核清单**

在使用AI生成的建筑专业文本时，建议按以下清单逐项审核：

| 审核项 | 检查方法 |
|--------|---------|
| 规范条号是否正确 | 对照规范原文逐一核实 |
| 数据是否准确 | 查阅官方统计数据或检测报告 |
| 案例是否真实 | 网络搜索核实项目是否存在 |
| 材料参数是否可靠 | 以材料供应商数据或检测报告为准 |
| 技术方案是否可行 | 请有经验的专业人员审核 |
| 是否存在安全隐患 | 重点核查涉及消防、结构安全的内容 |

#### 3.2 特别警告：高风险领域

以下建筑领域的AI生成内容**必须经过严格人工审核**，绝不能直接使用：

```
🚨 高风险领域：
├── 结构计算和荷载数据
├── 消防设计和疏散计算
├── 建筑材料的力学和热工参数
├── 施工安全技术方案
├── 工程造价数据
└── 法律法规引用

原则：涉及人身安全和法律责任的内容，AI只能辅助，不能决策。
```

- 要求：使用提示词模板，指定保留专业术语的英文原文
- 提交：翻译结果 + AI分析要点

**任务3：提示词对比实验（30分钟）**
- 选择一个题目（如"为某咖啡厅撰写室内设计说明"）
- 分别用以下三种方式向AI提问，记录并对比输出：
  - L1：一句话模糊提问
  - L3：包含场景信息的提问
  - L5：完整七要素提示词
- 提交：三种方式的AI输出对比 + 你的分析总结（300字）

**任务4：课堂分享（30分钟）**
- 每组选1人展示本组最佳提示词和AI输出成果

#### 实践四：建筑专业文本生成实战（第3-4课时）

**实践目标**：独立完成一份建筑专业文本的AI辅助生成、审核与修改

**实践任务**（根据专业方向选做一项）：

| 专业方向 | 任务 |
|---------|------|
| **城乡规划** | 为一个假设的社区公园编写规划设计说明（1500字） |
| **建筑设计** | 为一个假设的小型文化馆编写建筑设计说明（1500字） |
| **室内设计** | 为一个假设的民宿客房编写室内设计说明（1500字） |
| **园林景观** | 为一个假设的校园景观节点编写设计说明（1500字） |

**实践步骤**：

1. **拟定项目信息**（15分钟）
   - 确定项目名称、地点、面积、功能需求等基本信息
   - 可以参考实际项目或合理虚构

2. **编写提示词**（15分钟）
   - 使用本课时学到的提示词七要素
   - 使用对应的专业模板

3. **AI生成初稿**（15分钟）
   - 将提示词输入AI
   - 通过多轮对话优化输出

4. **专业审核与修改**（30分钟）
   - 使用"人工审核清单"逐项检查
   - 标注AI生成内容中的可疑信息
   - 修改不准确或不专业的表述
   - 加入个人的设计思考和创意

5. **最终定稿**（15分钟）
   - 整理格式
   - 完善细节
   - 标注哪些内容是AI生成、哪些是个人修改

6. **互评与交流**（30分钟）
   - 组内互换作品，相互评审
   - 重点评估：专业性、创意性、AI使用的恰当性
   - 填写互评表

**评分标准**：

| 评分维度 | 占比 | 说明 |
|---------|------|------|
| 提示词质量 | 20% | 是否运用了七要素，提示词是否专业有效 |
| 内容专业性 | 30% | 设计说明的专业深度和准确性 |
| 审核修改 | 25% | 是否有效识别了AI幻觉并做了专业修改 |
| 个人创意 | 15% | 是否在AI基础上加入了个人思考 |
| 格式规范 | 10% | 文档格式是否整洁规范 |

### 本章小结

| 知识点 | 核心内容 |
|--------|---------|
| AI对话基础 | 五个提问层次，角色扮演，格式指定，多轮深入 |
| 提示词七要素 | 角色、背景、事项、要求、样式、风格、结构 |
| 专业文本生成 | 设计说明、分析报告、景观文本、施工方案的模板与方法 |
| AI幻觉 | 编造事实、虚构引用、张冠李戴等六种表现 |
| 减少幻觉 | 标注不确定、提供参考、交叉验证、限制发挥、人工审核 |
| 核心原则 | AI是助手不是替代；涉及安全的内容必须人工审核 |
